Installation Guide

EU AI Act Compliance Management System™

Repository: RRVI™

Version: 1.0.2

1. Purpose

This guide explains how to deploy the EU AI Act Compliance Management System within an organization.

The objective is to implement a complete, documented and inspection-ready Compliance Management System.

2. Prerequisites

Before installation ensure that:

  • the organization has identified AI usage;
  • management approves the implementation;
  • an AI Governance Owner is appointed;
  • a GitHub repository is available;
  • document version control is established.

3. Repository Installation

Clone or download the repository.

The repository shall preserve the following structure:

``` README.md

01-ai-system-inventory/ 02-risk-classification/ 03-ai-literacy/ 04-prohibited-practices/ 05-high-risk-assessment/ 06-technical-documentation/ 07-human-oversight/ 08-data-governance/ 09-logging-monitoring/ 10-incident-reporting/ ```

4. Installation Sequence

Implement the modules in the following order:

  1. AI System Inventory
  2. Risk Classification
  3. AI Literacy
  4. Prohibited Practices
  5. High-Risk Assessment
  6. Technical Documentation
  7. Human Oversight
  8. Data Governance
  9. Logging & Monitoring
  10. Incident Reporting

Do not skip modules.

5. Module Installation

For every module complete:

  • Policy
  • Procedure
  • Work Instruction
  • Checklist
  • Register
  • Evidence
  • Declaration

Only continue after the current module is complete.

6. Initial Population

Populate:

  • AI Inventory
  • Risk Register
  • Technical Documentation
  • Human Oversight Register
  • Data Governance Register
  • Logging Register
  • Incident Register

7. Verification

Verify:

  • 10 modules exist;
  • 70 controlled documents exist;
  • all registers are populated;
  • evidence exists;
  • declarations are signed.

8. Internal Audit

Execute the internal audit using:

  • AUDIT_PLAN.md
  • INSPECTION_GUIDE.md

Resolve all findings before production use.

9. Production Status

The Compliance Management System is considered operational when:

  • all modules are complete;
  • management approval is obtained;
  • document version is released;
  • evidence is available.

10. Maintenance

Review the system:

  • annually;
  • after regulatory changes;
  • after major AI changes;
  • after significant incidents.
